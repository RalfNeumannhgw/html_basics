<a> - Verlinkung

Das HTML-Kommando <a> steht für "anchor" (Anker) und wird verwendet, um Hyperlinks zu erstellen – also Verknüpfungen zu anderen Webseiten, Dateien, E-Mail-Adressen oder bestimmten Stellen innerhalb eines Dokuments.

Code:

<p>Link zu einer Webseite:</p>
<a href="https://www.google.de">Google besuchen</a>

<p>Link zu einer E-Mail-Adresse</p>
<a href="mailto:name@example.com">E-Mail senden</a>

<p>Link zu einem Abschnitt auf der gleichen Seite:</p>
<a href="#abschnitt1>Zum Abschnitt 1</a>

<p>Und später im Dokument</p>
<h2 id="abschnitt1>Abschnitt 1</h2>



Ergebnis:


Link zu einer Webseite:

Google besuchen

Link zu einer E-Mail-Adresse:

E-Mail senden

Link zu einem Abschnitt auf der gleichen Seite:

Zum Abschnitt 1

Und später im Dokument:


Abschnitt 1

<applet> (veraltet) - Multimedia

<area> - Multimedia

Diese Bereiche können anklickbar sein und zu verschiedenen Links führen. area wird nicht alleinstehend genutzt, sondern immer innerhalb eines map-Elements, das dann mit einem Bild (img) verknüpft wird.

Code:

<img src="https://upload.wikimedia.org/wikipedia/commons/c/cf/Worldmap_LandAndPolitical.jpg" usemap="#weltkarte" alt="Weltkarte" style="width: 10%;">

<map name="weltkarte">>
<area shape="rect" coords="34,44,270,350" href="europa.html" alt="Europa">
<area shape="circle" coords="477,300,44" href="afrika.html" alt="Afrika">
<area shape="poly" coords="130,60,180,20,230,60,180,100" href="asien.html" alt="Asien">
</map>>

<article>

    <p>HTML5 ist die aktuelle Version der Hypertext Markup Language und bietet viele neue semantische Tags.</p>
    <footer>
        <p>Veröffentlicht am 30. April 2025 von Max Mustermann</p>
    </footer>
</article>

<basefont> (veraltet) - Text & Inhalt

Das HTML-Kommando <basefont> wurde früher verwendet, um die Standardschriftart, -größe und -farbe für den gesamten Text auf einer HTML-Seite festzulegen. Der Text auf der gesamten Seite würde damit in Arial, Größe 4 und blau dargestellt – sofern keine anderen Styles in den Elementen überschrieben wurden. Veraltet seit HTML 4.01 und entfernt in HTML5 <basefont> gilt als veraltet und wird nicht mehr unterstützt. Es wurde durch CSS ersetzt, das wesentlich flexibler und standardkonform ist.


<style>
body {
    font-family: Verdana;
    font-size: 16px;
    color: blue;
}
</style>

Code:

  <basefont face="Verdana" size="4" color="blue">
<p>Dieser Text ist blau, Größe 4 und in Verdana.</p>

Ergebnis:

Dieser Text ist blau, Größe 4 und in Verdana.

<bdi> - Text & Inhalt

Das HTML-Tag <bdi> steht für "Bi-Directional Isolation" (bidirektionale Isolierung). Es wird verwendet, um einen Textabschnitt von der bidirektionalen Textausrichtung zu isolieren. Das bedeutet, dass der darin enthaltene Text unabhängig von der globalen Textausrichtung auf der Seite behandelt wird, was besonders bei Sprachen mit unterschiedlicher Schreibrichtung (z. B. von rechts nach links und von links nach rechts) hilfreich ist.

<p>Hier sehen wir, wie der Text korrekt isoliert wird:</p>
 
<p> Ohne bdi: اسم المستخدم: user123 و اسم آخر: admin456</p>
<p> Mit bdi: اسم المستخدم:  <bdi>user123 </bdi> و اسم آخر: <bdi>admin456</bdi></p>

Ergebnis:

Beispiel für <bdi> im Body

<body> - Grundstruktur

Das HTML-Kommando <body> definiert den sichtbaren Inhaltsbereich einer HTML-Seite – also alles, was im Browserfenster angezeigt wird: Texte, Bilder, Links, Tabellen, Formulare, Skripte und mehr. Funktion von <body>: Enthält den gesamten Seiteninhalt, der für die Nutzer sichtbar/interaktiv ist. Ist ein Pflichtbestandteil eines HTML-Dokuments. Das <body>-Tag ist das Herzstück jeder Webseite. Es umfasst alles, was die Besucher deiner Seite sehen und benutzen. Muss genau einmal pro HTML-Dokument vorhanden sein.

<br> - Text & Inhalt

Das HTML-Element <br> steht für "line break" (Zeilenumbruch) und wird verwendet, um einen manuellen Zeilenwechsel innerhalb von Texten zu erzwingen – ohne einen neuen Absatz zu starten. Nicht übermäßig verwenden – <br> ist kein Ersatz für CSS-Layout oder Absatzstrukturierung (<p>, <div>, etc.). Bei langen Texten sollte man stattdessen lieber semantische Tags wie <p> oder <section> einsetzen.

Code:

<button> - Formulare

Das HTML-Element <button> wird verwendet, um einen interaktiven Button (Schaltfläche) auf einer Webseite zu erstellen. Es kann Klick-Aktionen auslösen – etwa zum Absenden eines Formulars, für JavaScript-Funktionen oder für eigene interaktive Zwecke.Vorteile gegenüber <input type="button">: <button> kann beliebigen Inhalt enthalten: Text, HTML, Symbole, Bilder. Folgende Attribute können gesetzt werden: submit (Standard in Formularen – sendet das Formular); reset (Setzt Formularfelder zurück); button (Nur Button, keine Standardaktion)

Code:

<form>
    <input type="text" name="name" placeholder="Dein Name">
    <button type="submit">Senden</button>

<canvas> - Multimedia

Das HTML-Element <canvas> ist ein Container für dynamische, scriptgesteuerte Grafiken. Es wird häufig verwendet, um 2D-Grafiken, Animationen, Diagramme oder sogar Spiele zu rendern. Der <canvas>-Tag allein stellt eine leere Zeichenfläche zur Verfügung, und die eigentliche Grafik wird mithilfe von JavaScript darauf gezeichnet. Zeichnen von Linien, Formen (Kreis, Rechteck, etc.); Grafiken und Animationen (z.B. Spiele, interaktive Grafiken); Diagramme und Visualisierungen (z.B. Balken-, Linien- oder Kreisdiagramme); Bildbearbeitung (z.B. Filter und Effekte)

Code:

<canvas id="meinCanvas" width="500" height="500"></canvas>

<script>
    const canvas = document.getElementById('meinCanvas');
    const ctx = canvas.getContext('2d');  // 2D-Kontext für Zeichnen
    
    // Zeichne ein Rechteck
    ctx.fillStyle = "blue";
    ctx.fillRect(50, 50, 150, 100);
    
    // Zeichne einen Kreis
    ctx.beginPath();
    ctx.arc(300, 300, 50, 0, 2 * Math.PI);
    ctx.fillStyle = "red";
    ctx.fill();
</script>

Ergebnis:

    <embed> - Multimedia

Zweck von <embed>: Betten externe Ressourcen direkt im Browser ein. Wird oft für Inhalte genutzt, die nicht nativ zu HTML gehören (z.B. PDF-Viewer, SVG, Flash – heute veraltet). Funktioniert als Container für nicht-HTML-Medieninhalte.     <i> - Text & Inhalt

    Das HTML-Kommando <i> steht für "italic" (kursiv) und wird verwendet, um Text kursiv darzustellen – typischerweise für Inhalte, die besonders hervorgehoben, aber nicht wichtig im Sinne der Betonung sind. Früher (in HTML4) wurde <i> rein für die Darstellung verwendet (also einfach Schriftschrägung). In HTML5 hat das Tag zusätzlich eine semantische Bedeutung: Es wird für Text verwendet, der aus stilistischen oder typografischen Gründen hervorgehoben wird, z. B.:

    • Fachbegriffe
    • Fremdwörter
    • Gedanken
    • Namen von Kunstwerken, Buchtiteln etc.

    Wenn du etwas inhaltlich betonen willst (nicht nur stilistisch), solltest du besser <em> verwenden.

    Code:

    <p>Das Wort <i>kursiv</i> ist in Schrägschrift dargestellt.</p>
<p>Das ist <em>wirklich</em> wichtig</p>

    <keygen> - Veraltet

    Das HTML-Tag <keygen> war ein veraltetes Element, das ursprünglich verwendet wurde, um ein Schlüsselpaar (Public/Private Key Pair) im Browser zu generieren und den öffentlichen Schlüssel an den Server zu senden, z. B. für die clientseitige Zertifikatsauthentifizierung.

    Dabei wurde:
    • Der öffentliche Schlüssel an den Server gesendet (im key-Feld).
    • Der private Schlüssel sicher im Browser gespeichert.
    • Diese Technik sollte den Aufbau einer Client-Zertifikatsinfrastruktur erleichtern.

    Das <keygen>-Element ist in HTML5 veraltet (deprecated) und wird nicht mehr von allen modernen Browsern unterstützt (z. B. Chrome, Firefox und Edge haben es entfernt). Sicherheitsbedenken und mangelnde Kontrolle über das Schlüsselmanagement im Browser führten zur Abkehr.

    Alternative:

    Für sichere Authentifizierung und Schlüsselaustausch verwendet man heute:

    • Web Cryptography API (clientseitige Krypto-Funktionen in JavaScript)
    • FIDO2/WebAuthn für passwortlose Authentifizierung
    • TLS-Clientzertifikate (außerhalb von HTML gesteuert)

    Zweck von <keygen>:

    Code:

  <input type="submit">
</form>
